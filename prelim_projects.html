1. Moving Least Squares

Objective:

Objective of this project is to make you understand how a point cloud is being smoothened or sampled to remove the noise that induced by the sensors like lasers, kinect, etc. You will implement MLS algorithm for 2D point cloud and do the below

  • Try different descreasing functions
  • Try different polynomial functions
  • Try different tricks of sampling initial data

At the end of this project you will know what parameters to use to smoothen a particular data.

Programming instructions:

For this project you are free to use any programming language. It should contain a GUI that can display the 2D points (both input, output). You are welcome to use libraries like OpenCV, Qt, Java, C++ or any language for this purpose. We will provide you with git-repo that has the input data files and also a basic stencil code in js that has a canvas that can show you the points from the data files. You are welcome to use this stencil code and build up on that.

Github link to the data and code:

github

Implementation details:

Totally you will implement 3 functions.

  1. get_rlist():

    Given a set of points you should be able to sample them in such a way that you approximate them. There are existing methods to do this which will be covered in the class, as part of presentations. But you are welcome to try your own ways and give the analysis. So this function outputs a list of points which we call rlist.

  2. get_qHlist():

    Given plist (points given as input) and rlist, this function is going to compute the local frame of reference and get the origin corresponding to the local point r in the list. So for each r you will get a q and a H (i.e. local line passing through q). Computing this for all the points in rlist will get you qlist and Hlist as the output of this method.

  3. get_q_list()

    Given qlist and Hlist, if you chose a polynomial (computed from optimization routine) will give you a point list which we call q_list would be the smoothened point cloud

2. RGBD Mapping

Objective:

Objective of this project is to make you get acquainted with 3D point cloud data, and place a sequence of point cloud frames using transformations from the robots odometry to build a RGBD map. At the end of this project you will know what are the issues with mapping techniques and what are the open research problems in this areas.

Programming instructions:

For this project you are supposed to use PCL a C++ library for processing point cloud data. You will be introduced to the data and the processing methods in a lab sessions. You will also have to show the results in the PCL visualizer which has the properties to place the point clouds spacially at the specified locations in 3D using transforms.

Implementation details:

Totally you will implement 3 functions.

  1. read_pcd_odom():

    You will be given a sequence of point cloud data (pcd files) along with the odomotry readings from the robot in a file. You will have to read the files and store them in a specific datastructure.

  2. compute_trasforms():

    This method will be the core of the project where you will compute transformations of the cloud viewpoints with respect to the odometry readings. Transformations will be covered in the class as a refresher.

  3. show_map()

    As you compute the transforms of the cloud corresponding to the odometry, you will have to place these clouds in a visualizer along with the computed transforms.
