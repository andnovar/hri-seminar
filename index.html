Introduction

CS1950-E is a seminar course on Human-Robot Interaction. The theme of this course in fall 2015 is going to be "Creating usability interactive maps from Robot sensing". Google maps that we use in our daily life is built using satelite images, street views and GPS data. To make use of this map, it has to be labelled properly making use of location information, GPS locations, routes and etc. Once it is labelled, algorithms are develop to plan a route between locations based on many constraints like the shortest path, fastest path, for public transport and so on. After having the planning module setup for all the constraints the google map is ready to be used. Similarly for a robot to move across the locations in a building autonomously it requires a map building phase followed by a labelling phase and then a planning phase. In this course we will focus on building maps using RGBD sensors followed by interactively labelling these maps so that they can be used to plan a task.
We will introduce you to the perceptual challenges involved in robotics when building a good map. In the process you will be going through some basics in robot sensing (from Odometry, Lasers, RGBD Sensors and Camera), mapping and state estimation problems like localization and SLAM. We will essentially discuss conference papers covering 5 topics such as 1) Smoothing RGBD data from Kinect, 2) RGBD Mapping, 3) History of SLAM in robotics (SGD, FastSLAM, EKF) 4) Optimization techniques used in mapping and SLAM (iSAM, ICP), and 5) Human interaction with the RGBD data . This course focuses on SLAM part of robotics and how human interaction on these maps would help in building better maps.

This year, CS1950-E course work will involve each individual discussing atleast one paper during the entire semester (ofcourse Faculty/TA will help you understand the technical details) and assignments will be to implement some standard techniques in each of the above mentioned topics. There will be a final project of implementing something substantial out of your assignments.

Projects and Grading

There will be 4 programming assignments and 1 main project. CS 1950E projects are graded as “checked” (completed) or “not checked” (incomplete). To receive an A grade in the course, you must have all assignments checked by the end of reading period. For a B grade, 3 assignments must be checked. Less than 3 checks will result in no credit for the course. The timing and due dates for these projects will be announced on an ongoing basis.

Final Grading

All grading will be finalized on TBA. Regrading of specific assignments will be done upon request during office hours. No regrading will be done after grades are finalized.

Repositories

The course staff will provide a git repository (through github) for each student group to check in project source code and reports, unless you request using separate service or repository. Please refer to this tutorial for an in-depth introduction to git and version control. Although it has a coarse language choice, this is also a great and accessible tutorial. We expect students to use these repositories for collaborative development as well as project submission. It is the responsibility of each student group to ensure their repository adheres to the Collaboration Policy and submission standards for each assignment. Submission standards and examples will be described for each assignment.

Late Policy

Do not be late submitting assignments. The course staff reserves the right to not grade late submissions.

Collaboration Policy

TBD This policy covers all course material and assignments unless otherwise stated. Course material and Turtlebot/ROS information and documentation may be discussed with anyone.
